# Agent المصنع - ZK Biometric Sync Agent

سكريبت يُشغَّل على كمبيوتر داخل المصنع، يتصل بأجهزة البصمة ZKTeco على الشبكة المحلية ويرسل بيانات الحضور إلى السيرفر البعيد.

---

## المتطلبات

- **Node.js** النسخة 16 أو أحدث — [تحميل من nodejs.org](https://nodejs.org)
- اتصال شبكي بين الكمبيوتر وأجهزة البصمة (شبكة محلية)
- اتصال إنترنت للوصول إلى السيرفر البعيد

---

## التثبيت

```bash
# 1. انسخ مجلد agent/ إلى الكمبيوتر
# 2. افتح الـ Terminal/CMD داخل المجلد
cd agent

# 3. ثبّت الحزم المطلوبة
npm install
```

---

## الإعداد

أنشئ ملف `.env` داخل مجلد `agent/` وضع فيه الإعدادات التالية:

```env
# عنوان السيرفر البعيد (بدون شرطة مائلة في النهاية)
SERVER_URL=https://your-server.com

# مفتاح API — احصل عليه من صفحة "Agent المصنع" في الموقع
AGENT_API_KEY=your_api_key_here

# قائمة الأجهزة مفصولة بفواصل
# الصيغة: IP:PORT:اسم-الجهاز:workshopId
# workshopId اختياري — معرّف الورشة من قاعدة البيانات
DEVICES=192.168.1.100:4370:ورشة-الخياطة,192.168.1.101:4370:ورشة-التشطيب

# مهلة الاتصال بالجهاز (بالميلي ثانية) — اختياري، الافتراضي 10000
TIMEOUT_MS=10000

# فترة التكرار التلقائي (بالدقائق) — يُستخدم فقط مع --watch
INTERVAL_MINUTES=30
```

### كيفية الحصول على معرّف الورشة (workshopId)

1. افتح الموقع وانتقل إلى **"Agent المصنع"**
2. المعرّف موجود في رابط الورشة أو يمكن الاستعلام عنه من قاعدة البيانات

## التشغيل

### تشغيل مرة واحدة

```bash
node zk-agent.js
```

### تشغيل دوري تلقائي (كل 30 دقيقة مثلاً)

```bash
node zk-agent.js --watch
```

---

## جدولة التشغيل التلقائي

### على Windows — باستخدام Task Scheduler

1. افتح **Task Scheduler** من قائمة ابدأ
2. أنشئ مهمة جديدة
3. في **Actions**: أضف `node C:\path\to\agent\zk-agent.js`
4. في **Triggers**: حدد الجدول الزمني المطلوب (كل ساعة مثلاً)

أو استخدم أمر PowerShell:

```powershell
$action = New-ScheduledTaskAction -Execute "node" -Argument "C:\agent\zk-agent.js" -WorkingDirectory "C:\agent"
$trigger = New-ScheduledTaskTrigger -RepetitionInterval (New-TimeSpan -Hours 1) -Once -At (Get-Date)
Register-ScheduledTask -TaskName "ZKAgent" -Action $action -Trigger $trigger -RunLevel Highest
```

### على Linux/macOS — باستخدام cron

```bash
# افتح محرر cron
crontab -e

# أضف هذا السطر لتشغيل كل ساعة
0 * * * * /usr/bin/node /home/user/agent/zk-agent.js >> /var/log/zk-agent.log 2>&1
```

## آلية العمل

1. يقرأ الـ Agent إعداداته من ملف `.env`
2. يتصل بكل جهاز بصمة على الشبكة المحلية
3. يجلب جميع سجلات الحضور
4. يُجمّع السجلات حسب الموظف واليوم (أول بصمة = دخول، آخر بصمة = خروج)
5. يرسل البيانات إلى endpoint `/api/agent/push-attendance` في السيرفر
6. السيرفر يحسب التأخير والخصومات ويحفظ البيانات

---

## استكشاف الأخطاء

| الخطأ | الحل |
|-------|------|
| `انتهت مهلة الاتصال` | تأكد أن الجهاز متصل بالشبكة وعنوان IP صحيح |
| `تم رفض الاتصال` | تأكد أن الجهاز يعمل والمنفذ 4370 مفتوح |
| `مفتاح API خاطئ` | تأكد من AGENT_API_KEY في .env أو جدّد المفتاح من الموقع |
| `ملف .env غير موجود` | أنشئ الملف داخل مجلد `agent/` |
| `DEVICES غير محدد` | أضف قائمة الأجهزة في .env |
